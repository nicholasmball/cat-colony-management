Notification centre — nav badge & /app/notifications list

A Notifications nav entry with an unread badge opens a full page (decision A) listing all the manager's alert rows newest-first. Freshness is on-load (a); read is explicit — per-item and a "Mark all read" (ii). Every pattern is reused from the dashboard and incidents list: the card token, the border-l-4 urgent/concern accents, the icon+text severity convention from incident-status-pill, EmptyState, ChevronIcon, and the bilingual copy from the alert catalog.

Option 1 — the page shows every alert row regardless of channels. Even an urgent alert whose dispatch intent was push+SMS appears here, so the in-app centre is a complete, trustworthy record — nothing urgent is hidden behind a channel. Clicking a row navigates to the colony / cat / incident and marks it read.

1Nav unread badge — desktop sidebar & mobile tab bar

A new { href: "/app/notifications", labelKey: "nav.notifications" } item joins the manager nav in lib/nav-items.ts. The badge is a pill carrying a small bell glyph and the count (icon + number — never colour alone), capped at 9+, and hidden entirely when 0. It is keyboard-focusable as part of the nav link and announced as aria-label="3 unread notifications".

2/app/notifications — populated (mixed, all 6 types)

Header = "Notifications" + a "Mark all read" action (shown only while there are unread). Rows are newest-first. Each row: a severity indicator (▲ urgent / ● routine — icon + text), the localized title (bold) + a one-line body from the alert catalog, a locale-aware relative time, a chevron showing it links onward, and a per-row mark-read control. Unread rows get a left accent bar + an unread dot + bolder title; read rows are muted.

Accessibility checklist (WCAG 2.1 AA)

  • List semantics. <ul role="list"> of <li> rows; the page header is an <h1>, the count an honest sub-line.
  • Never colour-alone. Severity = ▲/● glyph + "Urgent"/"Routine" word; unread = left-bar shape + unread dot + bolder weight, not tint alone (1.4.1).
  • Badge announced. Nav badge has aria-label="N unread notifications" ("9 or more…" at the cap); hidden when 0 so screen readers aren't told "0 unread".
  • Labelled controls. Per-row mark-read button is named for its alert ("Mark “…” as read"); read rows expose a disabled ✓ "already read". "Mark all read" is labelled and only present with unread items.
  • Focus order & targets. Tab order = header action → each row (row link, then its mark-read button); every interactive target ≥ 44px (the mark-read hit area is 44×44 even though the glyph is small); visible focus-visible:ring-accent/25 reused from the app.
  • Click = navigate + mark read. Activating the row body opens the entity and marks it read; the mark-read control marks read without navigating, so a manager can clear noise without leaving the list.
  • Contrast. All text ≥ 4.5:1 (titles on white, muted body, red-ink on red-bg for the urgent tag) — matches the tokens already audited on the dashboard.
  • 375px & desktop. Same row composition at both widths; title truncates with ellipsis, body is single-line, the mark-read + chevron stack stays a fixed-width aside so nothing reflows awkwardly.
