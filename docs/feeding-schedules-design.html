Feeding schedules

Design for the Feeding schedules feature. Building on the approved Requirements & Acceptance Criteria step: caretakers/admins assign feeders to a colony on a weekly or one-off basis, and those assignments power the feeder's /app/today list. The schedule lives as a new section on the colony detail page (/app/colonies/[id]), directly below Cats; the add/edit form is its own mobile-friendly page. Feeders see the schedule read-only; managers get full CRUD.

1. Flow

Caretaker opens a colony → scrolls to Feeding schedule → taps Add schedule → picks a feeder, chooses Weekly (taps the weekdays) or One-off (picks a date), optionally sets an approx time + note → Save → back on the colony, the new row(s) appear under the schedule. The next morning the assigned feeder opens Today and sees only the colonies they're scheduled to feed; everyone else's colonies have dropped away. No modals — full pages keep back-navigation honest in the field and offline-safe.

Reuse: the schedule section mirrors the existing Cats section on the same page — a muted uppercase heading + count, a manager-only Add button, row cards, and the shared EmptyState when there's nothing yet. The form reuses input / fieldLabel / btnPrimary from lib/ui.ts; delete reuses ConfirmButton.

2. Mobile screens

Four states: the manager's schedule section (rows + Add), the Add/Edit form in both its Weekly and One-off shapes, the feeder's read-only schedule view, and the new Today "no assignments" empty state.

4. Accessibility & "when" vocabulary

  • Weekday toggles are real buttons — each "M T W T F S S" cell is a role="button" with aria-pressed, grouped in a role="group" labelled "Repeats on". Operable with Tab to enter the group, arrow/Tab between days, and Space/Enter to toggle. A clear 2px focus ring (offset) shows the current cell. Each is exactly 44px tall.
  • Type toggle is a labelled segmented control — role="group" "Schedule type", Weekly / One-off as the two options; selection shown by fill and the word, never colour alone. Only the matching control (weekdays or date) renders, so there's never an ambiguous half-filled form.
  • Every field has a real <label> (Feeder, Date, Approx time, Notes) via fieldLabel; optional fields say "(optional)" in the label, not by absence.
  • ≥44px targets throughout — feeder select, date/time inputs, day toggles, Save/Cancel and the per-row edit/delete icon buttons (34px visual + padding to a 44px hit area).
  • Delete via ConfirmButton — the 🗑 is a submit inside a server-action form that asks "Remove this schedule?" before deleting; it's an icon and an aria-label="Delete schedule for ana@scot.pt", never icon-only to assistive tech.
  • Errors use role="alert" — e.g. "Pick a date to save this one-off feed" or "Choose at least one weekday" announce immediately and sit next to the offending control.
  • "When" never colour-alone — the ⟳ weekly / ★ one-off pills carry a glyph and a word; the human-readable day list ("Mon, Wed, Fri" or "Sat 14 Jun") is the real content. Legible in greyscale.
  • Feeder identity is the email — minimal personal data by design (GDPR decision); the picker and rows show email, no names or phone numbers.

For the engineer: a weekly multi-day selection writes one DB row per weekday — "Mon/Wed/Fri" creates 3 feeding_schedules rows (weekday 1,3,5), so Today's filter is a simple weekday = todayWeekday match with no array logic. A one-off is one row with a scheduled_date. Approx time is advisory only — it's shown on the row and to the feeder, but feeding status (fed/pending/missed) still derives from the colony's feeding window, exactly as in the Feeder-Today card. Edit of a weekly schedule re-syncs the weekday rows for that feeder; all writes are org-scoped (organisation_id) and RLS-guarded.

5. Mapping to acceptance criteria

  • Feeding-schedule section on /app/colonies/[id] below Cats, rows showing feeder email · when · approx time · notes → §2 manager screen, §3 desktop.
  • Managers (admin/caretaker) see Add schedule + per-row edit/delete; feeders see it read-only → §2 manager vs feeder screens, §4 (omit not disable).
  • Add/Edit form is a mobile-friendly page (/app/colonies/[id]/schedules/new & …/[sid]/edit), no modal → §1 flow, §2 form screens.
  • Feeder picker = org members with role feeder/caretaker, shown by email → §2 Feeder field, §4 email-only note.
  • Type toggle Weekly vs One-off; Weekly → 7 keyboard weekday toggles (multi-select), One-off → date input; only the relevant control shows → §2 both form states, §4 toggle bullets.
  • Optional approx time + optional notes → §2 form fields (labelled "(optional)").
  • Weekly multi-day = one row per weekday; approx time advisory only (status uses colony window) → §4 engineer note.
  • Today filtered to the feeder's colonies for today; empty → EmptyState "No feeds assigned to you today" with the given body → §2 Today screen + before/after note.
  • Accessible: weekday keyboard/focus, real labels, ≥44px, delete via ConfirmButton, errors role="alert", when never colour-alone → §4.
