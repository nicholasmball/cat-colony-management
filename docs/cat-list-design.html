Cat list & cat detail

Design for the Cat list (per colony) card. Building on the Requirements & Acceptance Criteria step: the list already exists on the colony page — the work is making every row a tappable link to a cat detail page for all roles (Feeders included), and delivering that lightweight detail page as the destination.

1. Flow

Colony → tap a cat row → Cat detail (read-only) → back to colony, or (managers only) Edit. One tap from the feed view to a cat; one tap back. No modals — full pages keep context and back-navigation honest on mobile.

3. Interaction & accessibility notes

  • Whole row is the link (Fitts's Law) — a 60px-tall <Link>, well above the 44px touch minimum. The old manager-only "Edit" corner link is removed from the row; Edit now lives on the detail page so every role taps the same target.
  • Chevron (›) as information scent — signals "this row goes somewhere," not just a static list item. Reinforces the tap affordance without relying on colour.
  • Status uses pill + word, never colour alone (WCAG 1.4.1). Green "seen", amber "concern", red "not seen" each carry a text label.
  • Detail hero photo uses the same presigned GET as the thumbnail; paw fallback when no photo. alt = "Photo of {name}".
  • Sparse-data rule — only populated fields render. Name falls back to description falls back to "Unnamed cat". Never blocks render on a missing field.
  • Back navigation returns to the colony, preserving scroll where the platform allows.
  • Loading — list and detail are server-rendered; thumbnails resolve in one batched Promise.all (no per-row waterfall). No spinner needed for first paint.

4. Status vocabulary (from sightings)

Sighting status Pill Meaning on the row/detail
seen seen Recorded at the latest feed
concern concern Feeder flagged something off
not_seen not seen Absent — never auto-promoted to "missing" (human review owns that)
— active No recent sighting yet; cat record's base status

This card surfaces the cat's current status only. The full status history timeline is intentionally deferred to the "Cat detail & status history" card — this design gives that feature a page to grow into without redoing the layout.

5. Mapping to acceptance criteria

  • Tappable row → cat detail, all roles → screens 1–2.
  • Detail renders photo/name/colour/status + optional fields, omits missing → screens 2–3.
  • Manager Edit affordance, Feeder read-only → screen 3.
  • Org-scoped, 404 on cross-org/soft-deleted → server query with .is("deleted_at", null) + RLS → notFound().
  • Mobile-first, ≥44px targets, batched thumbnails → interaction notes.
