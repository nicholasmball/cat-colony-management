Caretaker Dashboard — UX Design

A read-only manager oversight roll-up at /app/dashboard. Answers the four daily questions at a glance — Was the colony fed? Which cats were seen? Are there new or missing cats? Is there a problem? — and surfaces the "act now" items first. Managers land here after login; feeders never see it.

Visual language matched to app/today, app/incidents and the colony "Cats of concern" section. Mobile @375px and desktop @≥768px. Every state drawn: populated, empty/all-clear, loading skeleton, whole-page all-clear.

Hand-off & scope: every drill-in is a link into an existing page (Today, the feed flow, the cat-confirm page, the incident detail, the cat page) — the dashboard itself stays read-only and adds no new actions. Reuses the existing detection (feeding-status, concern-candidate, urgency levels). Alert fan-out (push/SMS) is a separate card and out of scope here.

1 · Desktop (≥768px) — populated, full dashboard

Two-column grid uses the width instead of stranding a single mobile column. Act-now cards (Missed feeds, Urgent incidents) lead with a red left-border; Today's-feeds summary spans both columns as the daily anchor.

5 · Nav placement & landing

Dashboard is the first item in the manager nav (before Home/Today), shown only to admins + caretakers — navItemsFor already gates manager items, so feeders never see it. Managers land here post-login (the /app home redirect for managers points to /app/dashboard; feeders continue to land on /app/today). A new grid/squares glyph is added to iconByHref for the /app/dashboard entry, in the existing icon family (viewBox 24, currentColor).
