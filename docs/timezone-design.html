Timezone & "today"

Building on the approved Requirements (org-level timezone, default Europe/Lisbon): this card is 90% backend — a migration, a tested lib/time.ts, and routing day math through it. The only new pixels are one field in Org settings. Designing that field + the mental model so the value is legible.

1. The only new UI — Org settings → Timezone

Adds a single <select> to the existing /app/org form (admin-only, already gated). Recognition over recall: the current zone is pre-selected and an inline hint shows the live local date so the admin can confirm it's right without leaving the page.

Short list keeps the decision fast (Hick's Law). If an org's stored zone isn't listed, it's added to the top so it's never lost.
Accessibility: native <select> (keyboard + screen-reader friendly, zero JS), a real <label>, the hint is text not colour, the field meets 44px. Save reuses the existing success/error banners on the page.

2. The mental model — where "today" begins

The thing volunteers must trust: "today" and "missed" follow Tavira's clock, not the server's. A feed logged at 00:30 local belongs to the new day even though the server (UTC) may still call it yesterday.

Day boundaries (00:00–24:00) are computed in the org zone and converted to UTC instants for querying. "Missed" fires 12h after the local window close, also in-zone — so it survives DST and a UTC server.

3. Downstream (not built here, shown for context)

Feeder "Today", Feeding schedules and the missed-feed alert are separate cards; they'll read these helpers. This card just guarantees that when they ask "is it today?" / "is this late?", the answer is in the right timezone and unit-tested — including a non-Portugal org.

4. Acceptance-criteria coverage

  • Admin can view + change the org timezone in /app/org → §1 field, pre-selected + hint.
  • Legibility / recognition → inline "currently HH:MM, day" confirmation.
  • Stored-but-unlisted zone never lost → pinned to top of the list.
  • Backend tz-correctness + tests → §2 model; no UI needed, no new states beyond the existing save banners.
