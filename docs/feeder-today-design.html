Feeder Today

Design for the new /app/today screen — the feeder's daily landing. Building on the Requirements & Acceptance Criteria step: one glance answers "was the colony fed?" and one tap starts the 30-second feeding update. The screen lists all active colonies in the active org (owner-decided scope — the "only my assignments" filter is deferred to the Feeding-schedules card), ordered so the work that still needs doing sits at the top.

1. Flow

Open app → (feeder lands on) Today → scan statuses → tap a pending or missed row → quick Feed update → save → back to Today, where that row now reads ✓ fed · <time> and drops to the bottom. A fed row instead offers View → into the colony. No modals — full pages keep back-navigation honest in the field.

Landing rule: a feeder hitting /app is redirected to /app/today; Admins and Caretakers keep their current home. "Today" is also a primary nav tab (calendar icon) for everyone, so managers can still open it.

4. Accessibility & status vocabulary

  • Icon + word, never colour alone (WCAG 1.4.1) — every status is a glyph and a label: ✓ fed ● pending ⚠ missed. The list is fully legible in greyscale.
  • Contrast ≥4.5:1 — pill text on its tinted background and the accent action verb on surface all clear AA. Tones are a hint, not the message.
  • Whole row is the link, ≥60px tall — well above the 44px touch minimum; works at 320–375px without horizontal scroll.
  • No disabled buttons — feeders who can't add colonies get an empty state with explanatory copy and no CTA, not a greyed-out button.
  • Real headings & group labels — "Today" is the page <h1>; "Needs feeding" / "Done today" are real subheadings that double as screen-reader landmarks.
  • Status glyphs are aria-hidden — the adjacent word carries meaning, so assistive tech reads "missed", not "warning sign missed".
  • Timezone honesty — "today" is computed in the org's timezone, so a feed at 23:50 counts for the right day everywhere.
Status Row badge Applies when Row action
fed ✓ fed A feeding has been logged today (org tz). Shows the time, e.g. 07:42. View →
pending ● pending Not fed yet and (no window, OR window not closed, OR closed <12h ago). Feed →
missed ⚠ missed Not fed and the feeding window closed ≥12h ago (the alert threshold). Feed →
Ordering: actionable first — missed, then pending (by window start, no-window colonies last within the group) — then fed at the bottom. The feeder's eye lands on what still needs doing.
Pure status logic lives in feedingStatus({ fed, minutesAfterClose }) → "fed" | "pending" | "missed" — a side-effect-free function the engineer will unit-test (fed wins; else ≥720 min after close ⇒ missed; else pending). The screen only renders the result.

5. Mapping to acceptance criteria

  • New /app/today lists all active colonies in the active org → screens in §2–3.
  • Header "Today · <local date>" in the org timezone → §2 header, §4 timezone note.
  • Per-row name + feeding window ("08–09" / "no window") + status → row markup throughout.
  • Three statuses (fed / pending / missed) with the exact threshold rules → §4 table + feedingStatus.
  • Status is icon + word, never colour alone → §4 first bullet.
  • Row action: fed → "View →" (colony); pending/missed → "Feed →" (/app/colonies/[id]/feed) → row actions.
  • Ordering: actionable first (missed, then pending by window start), fed last → §4 ordering note.
  • Empty state reuses EmptyState ("No colonies to feed yet", no CTA) → §2 second phone.
  • "Today" is a primary nav tab (calendar) + the feeder's default landing; managers keep their home → §1 landing rule, §3 left nav.
  • "Only my assignments" filter explicitly deferred to Feeding-schedules → §3 note.
